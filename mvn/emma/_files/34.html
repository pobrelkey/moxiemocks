58/**
59 *
60 * Static class whose methods magically signal Moxie to flexibly match parameters of a mock expectation.
61 * <p>
62 * &nbsp;
63 * <h3>What are these methods?</h3>
64 *
65 * On certain method calls where you specify values that Moxie should expect as part of observed behavior -
66 * for example, method parameters on mock object calls, or return values on calls to spy objects - you can
67 * use the methods on <code>MoxieMatchers</code> to set flexible criteria for each parameter.
68 * <p>
69 *
70 * These are used to generate <a href="http://code.google.com/p/hamcrest/">Hamcrest</a> {@link Matcher}s
71 * which will be matched against the actual value(s).
72 * <p>
73 * &nbsp;
74 * <h3>How do they work?</h3>
75 *
76 * (The gory details of the magic behind this class follow - you may safely skip to the next section
77 * if this bores/confuses you.)
78 * <p>
79 *
80 * Every time you call a method on <code>MoxieMatchers</code>, a <code>Matcher</code> is pushed onto a stack,
81 * along with (in many cases, especially for primitives) an expected parameter type.  The method then returns
82 * the default value (<code>null</code> for methods returning an object, zero/<code>false</code> for methods
83 * returning a primitive value).
84 * <p>
85 *
86 * When you call a method that can take matcher invocations as parameters and the matcher stack is non-empty,
87 * the arguments to that method are examined from right to left.  For arguments that are equal to the default value,
88 * the <code>Matcher</code> on top of the stack is popped off and used (unless it expects a value of an
89 * incompatible type, in which case an error is raised).  For non-default arguments, an
90 * {@link org.hamcrest.core.IsEqual IsEqual} matcher is created which will match on a value equal to that argument.
91 * <p>
92 *
93 * When you call a method that can take matcher invocations as parameters and the matcher stack is empty,
94 * an {@link org.hamcrest.core.IsEqual IsEqual} matcher is created for each parameter.
95 * <p>
96 *
97 * At the end of this process, the matcher stack is checked, and an error is raised if it is non-empty.
98 * (Note that this does not occur in methods like {@link #and(Object[]) and()}, {@link #or(Object[]) or()}
99 * and {@link #not(Object) not()} which may themselves be part of matcher expressions.)
100 * <p>
101 * &nbsp;
102 * <h3>What do I need to remember?</h3>
103 * <ul>
104 * <li>Never save the result of a call to <code>MoxieMatchers</code> to a variable - always plug it straight in to where it needs to go.</li>
105 * <li>If you're using a <code>MoxieMatchers</code> call anywhere in a method call, make sure all zero/<code>false</code>/<code>null</code>
106 * parameters to that call are replaced with {@link #eq(int) eq(0)}, {@link #eq(boolean) eq(false)} or {@link #isNull()} respectively.</li>
107 * </ul>
108 */
